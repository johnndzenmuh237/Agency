Forge & Co — Agency Pro Template

A clean, professional, fully static HTML/CSS/JS template for an agency or studio website. No build tools, no frameworks, no dependencies beyond three Google Fonts loaded via CDN. Open any page directly in a browser or upload the whole folder to any static host.

Design System

Colors and type are defined once as CSS variables at the top of style.css, so a full re-theme only touches one block.

--paper --ink --copper --teal

Typography: Big Shoulders for headings (bold, condensed, uppercase), Inter for body copy, and IBM Plex Mono for labels, tags, and eyebrows.

Customizing Colors & Fonts

Open assets/css/style.css and edit the :root block near the top:

:root{
  --paper: #ECEAE3;   /* light background */
  --ink:   #1B1815;   /* primary text / dark sections */
  --copper:#C1542C;   /* primary accent / CTAs */
  --teal:  #2C6E6B;   /* secondary accent */
}

To change fonts, edit the @import at the very top of the same file and update the --font-display, --font-body, and --font-mono variables.

Replacing Placeholder Images

Every photo slot currently renders a styled .img-placeholder block (a diagonal pattern + label) instead of a broken image link. To drop in a real photo, replace the placeholder div with an <img> tag inside the same wrapper — sizing is already handled by the wrapper class:

<div class="team-photo">
  <img src="assets/images/team/mara.jpg" alt="Mara Whitfield">
</div>

JavaScript Modules

main.js

Runs on every page. Handles the sticky header shadow, mobile nav toggle, scroll-reveal animations ([data-reveal]), the FAQ accordion, animated stat counters ([data-count]), the back-to-top button, and the footer year stamp.

slider.js

Powers any element marked [data-slider] — used for the testimonial carousel. Supports arrows, dots, swipe gestures, autoplay, and pauses automatically when scrolled off screen.

portfolio-filter.js

Wires up .filter-btn[data-filter] controls to show/hide .portfolio-item[data-category] elements on the Portfolio page.

contact-form.js

Validates required fields and email format, then simulates a network submission with a loading state and success/error message. Replace the simulateSubmit() function with a real fetch() call to your backend or form service.

Adding a New Page

Copy any existing page closest to what you need, keep the <header> and <footer> blocks as-is (just update the active nav link), and replace the content inside <main>. All shared components — buttons, cards, the accordion, the form — are already styled and ready to reuse.

Browser Support

Built with standard, modern CSS (Grid, custom properties, clamp()) and vanilla JS (no jQuery, no build step). Tested against current Chrome, Firefox, Safari, and Edge. Layouts respond down to 360px-wide phones, and all animation respects prefers-reduced-motion.
